# 积分小管家 - 部署指南（Mac 版）

> 面向零基础用户，每一步都有详细说明。跟着做就行。

---

## 第一步：注册微信小程序账号（10 分钟）

### 1.1 打开注册页面

在浏览器打开：https://mp.weixin.qq.com/

点击右上角「立即注册」

### 1.2 选择账号类型

选择「小程序」→ 点击「去注册」

### 1.3 填写邮箱

- 输入一个**从未绑定过**微信公众号/小程序的邮箱（QQ邮箱、163邮箱、Gmail 都行）
- 点击「激活」，去邮箱里点击激活链接
- 回到页面继续

### 1.4 选择主体类型

- 选择「个人」
- 用你的微信扫码验证身份
- 填写姓名、身份证号等信息
- 完成注册

### 1.5 记录 AppID

注册完成后，用微信扫码登录微信公众平台。

进入「开发管理」→「开发设置」，页面上会显示 **AppID**，长得像这样：`wx1234abcd5678efgh`

**把它记下来，后面要用。**

---

## 第二步：安装微信开发者工具（5 分钟）

### 2.1 下载

在浏览器打开：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

选择 **Mac** 版本：
- Intel 芯片（老款 MacBook）选「macOS (Intel)」
- M1/M2/M3/M4 芯片（新款 MacBook）选「macOS (Apple Silicon)」

> 不知道自己是什么芯片？点屏幕左上角  →「关于本机」→ 看「芯片」那行

### 2.2 安装

- 下载完成后，双击打开 `.dmg` 文件
- 把「微信开发者工具」图标拖到「Applications」文件夹
- 从「启动台」或「应用程序」文件夹打开「微信开发者工具」
- 如果提示「无法验证开发者」：去「系统设置」→「隐私与安全性」→ 拉到最下面 → 点击「仍要打开」

### 2.3 登录

打开开发者工具后，用你的微信扫码登录。

---

## 第三步：配置项目（5 分钟）

### 3.1 填写 AppID

用任意文本编辑器打开项目里的这个文件：

```
child/project.config.json
```

找到 `"appid": ""` 这一行，改成你的 AppID：

```json
"appid": "wx1234abcd5678efgh"
```

保存文件。

### 3.2 用开发者工具打开项目

1. 打开微信开发者工具
2. 点击「+」号（导入项目）
3. 目录：点击「选择」，找到你的 `child` 文件夹，点击「打开」
4. AppID：应该自动填好了（如果没填，手动粘贴）
5. 后端服务：选择「微信云开发」
6. 点击「确定」

---

## 第四步：开通云开发（3 分钟）

1. 开发者工具顶部工具栏，找到「云开发」按钮，点击它
2. 首次使用会弹出开通引导：
   - 输入环境名称：随便填，比如 `prod`
   - 点击「确定」
3. 等待环境创建完成（约 30 秒）
4. 创建完成后，**记住环境 ID**（显示在云开发控制台顶部，类似 `prod-xxxxx` 这样的格式）

### 4.1 配置环境 ID

打开 `miniprogram/app.js`，把 `wx.cloud.init` 改成：

```javascript
wx.cloud.init({
  env: 'prod-xxxxx',  // 换成你的环境 ID
  traceUser: true
})
```

保存。

---

## 第五步：创建数据库（5 分钟）

1. 在开发者工具中点击顶部「云开发」按钮，打开云开发控制台
2. 点击左侧「数据库」
3. 点击「+ 添加集合」，依次创建以下 4 个集合：

| 集合名 | 说明 |
|--------|------|
| `users` | 用户信息 |
| `families` | 家庭信息 |
| `tasks` | 任务模板 |
| `records` | 积分流水 |

4. 每个集合都要设置权限。点击集合名 → 右上角「数据权限」→ 选择「所有用户可读，仅创建者可读写」→ 保存

> 不用纠结权限设置，所有数据库操作都是通过云函数完成的，这个权限只是兜底保护。

---

## 第六步：部署云函数（5 分钟）

云函数是小程序的后端逻辑，需要上传到云端才能运行。

1. 在开发者工具**左侧文件树**中，找到 `cloudfunctions` 文件夹并展开
2. 你会看到 5 个文件夹：`login`、`family`、`task`、`record`、`stats`
3. **逐个部署**：
   - 右键点击 `login` 文件夹
   - 选择「上传并部署：云端安装依赖」
   - 等右下角提示「上传成功」
   - 然后右键 `family` → 上传并部署
   - 然后 `task` → 上传并部署
   - 然后 `record` → 上传并部署
   - 最后 `stats` → 上传并部署

> 5 个全部部署完大约需要 2-3 分钟。

---

## 第七步：本地测试（5 分钟）

1. 点击开发者工具顶部的「编译」按钮
2. 模拟器中会出现小程序界面
3. 首次打开会显示「还没有加入家庭」
4. 点击「创建家庭」→ 输入家庭名称和孩子姓名 → 点击创建
5. 创建成功后首页显示积分和 8 个预设任务
6. 随便点一个任务试试记录功能

> 本地测试通过后，就可以进入下一步了。

---

## 第八步：体验版——给家人用（10 分钟）

体验版不需要上架审核，扫码就能用，适合给家人用。

### 8.1 上传代码

1. 点击开发者工具顶部的「上传」按钮
2. 弹出对话框：
   - 版本号：填 `1.0.0`
   - 项目备注：填 `初始版本`
3. 点击「上传」
4. 等待上传完成

### 8.2 设置为体验版

1. 在电脑浏览器打开 https://mp.weixin.qq.com/ 并登录
2. 左侧菜单 →「管理」→「版本管理」
3. 在「开发版本」那一行，点击「选为体验版」
4. 会生成一个**体验版二维码**

### 8.3 添加体验成员

家人不能直接扫码用，需要先添加为体验成员：

1. 在微信公众平台左侧菜单 →「管理」→「成员管理」
2. 在「体验成员」区域，点击「添加」
3. 输入家人的微信号，搜索到后添加
4. 可以添加最多 10 个人

### 8.4 家人使用

1. 把体验版二维码发给家人（截图发微信就行）
2. 家人用微信扫码打开
3. 家人点击「加入家庭」→ 输入你的邀请码
4. 加入后就能一起记录积分了

> 你自己的邀请码在小程序里：首页右上角 ⚙️ → 家庭管理 → 邀请码

---

## 第九步：正式上架——所有人可用（可选）

如果以后想开放给其他家庭使用，走正式上架流程。

### 9.1 补充小程序信息

在微信公众平台：

1. 左侧「设置」→「基本设置」
2. 上传小程序 Logo（随便做个图）
3. 填写小程序名称，如「积分小管家」
4. 填写小程序简介
5. 选择服务类目：
   - 个人主体可选「工具」→「备忘录」或「工具」→「信息查询」
   - 如果不确定选什么，选「工具 → 效率」一般都行

### 9.2 提交审核

1. 在微信公众平台左侧 →「管理」→「版本管理」
2. 找到「开发版本」，点击「提交审核」
3. 填写审核信息：
   - 选择刚才设置的服务类目
   - 功能页面：填 `pages/index/index`
   - 测试备注：「家长管理孩子积分的小工具，通过完成任务赚取积分」
4. 提交

> 审核一般需要 1-7 天，通常 1-2 天就能通过。个人主体的审核相对简单。

### 9.3 发布上线

审核通过后：

1. 微信公众平台会发通知（公众号消息通知）
2. 进入「版本管理」→「审核版本」
3. 点击「发布」
4. 选择「全量发布」

发布后，任何人都可以在微信里搜索你的小程序名称找到并使用。

## 常见问题

**Q: 打开开发者工具提示「当前项目未关联 AppID」**
A: 检查 `project.config.json` 里的 `appid` 是否填写正确。

**Q: 编译后白屏或报错**
A: 打开开发者工具底部的「调试器」→「Console」，看红色错误信息。最常见的是环境 ID 没配对。

**Q: 云函数部署失败**
A: 确认云开发已开通。如果提示环境问题，检查 app.js 里的环境 ID。

**Q: 点击任务报错「调用失败」**
A: 云函数可能没部署成功。重新右键每个云函数 →「上传并部署：云端安装依赖」。

**Q: 拍照上传失败**
A: 模拟器不支持真实拍照。用手机扫码「预览」版测试。

**Q: 体验版二维码家人扫码打不开**
A: 确认家人已添加为「体验成员」（第八步 8.3）。

**Q: 审核被拒**
A: 常见原因：小程序信息不完整、类目不匹配。按拒绝原因修改后重新提交。

**Q: Mac 上找不到项目文件夹**
A: 打开 Finder（访达）→ 菜单栏「前往」→「前往文件夹」→ 输入你的项目路径。
